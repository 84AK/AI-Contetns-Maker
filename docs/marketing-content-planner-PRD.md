# 마케팅 콘텐츠 기획 플래너 — 앱 기획서

> **브랜드**: AK LABS  
> **목적**: 바이브코딩의 5단계 PRD 기획 방식처럼, 마케팅 콘텐츠도 단계별로 쉽게 입력하고 기획할 수 있는 교육용 웹앱  
> **형태**: 단일 HTML 파일 (Single-file HTML/CSS/JS)

---

## 앱 개요

소상공인, 마케터, 학생 누구나 AI를 활용해 마케팅 콘텐츠를 단계별로 기획할 수 있는 웹앱.  
리서치부터 최종 AI 프롬프트 생성까지 5단계 흐름으로 완성하며, 결과물로 **PRD 기획서 + 스토리보드 + AI 제작 프롬프트**를 제공한다.

---

## 5단계 기획 흐름

바이브코딩 앱(`vibe-coding-planner.html`)의 5단계 구조와 1:1 대응:

| 바이브코딩 앱 | 마케팅 콘텐츠 기획 앱 |
|---|---|
| Step 1 · 문제 정의 | Step 1 · 리서치 및 문제 정의 |
| Step 2 · 사용자 정의 | Step 2 · 주제 선정 및 템플릿 선택 |
| Step 3 · 기능 설계 | Step 3 · 상세 구성 및 제작 가이드 |
| Step 4 · 화면 구성 | Step 4 · 스토리보드 미리보기 |
| Step 5 · 앱 이름 | Step 5 · 프롬프트 생성 및 기획서 완성 |

---

## 단계별 상세 설계

### Step 01 · 리서치 및 문제 정의

**목적**: 콘텐츠 기획의 출발점인 문제를 데이터 기반으로 정의한다.

**입력 항목**:

- **자료 수집** — 뉴스 기사 / 블로그 / 영상 / 웹페이지 URL 또는 텍스트 직접 입력  
  - 자료 출처 카테고리 버튼 (기사 / 영상 / 블로그 / 직접입력) 클릭 선택  
  - "AI로 요약하기" 버튼: 입력한 URL·텍스트에서 핵심 문제 자동 추출
- **핵심 문제 정리** — 수집한 자료들이 공통으로 가리키는 핵심 문제 서술  
  - 힌트: "~를 보고 ~해서 불편/안타깝다" 형식 안내
- **문제 해결 방향** — 이 문제를 콘텐츠로 어떻게 해결할 수 있을지 탐구 입력

**UI 패턴**: `hint-box` 가이드 + `textarea` + AI 자동 채우기 버튼

---

### Step 02 · 주제 선정 및 템플릿 선택

**목적**: 해결 방향을 바탕으로 최종 콘텐츠 주제를 확정하고, 형식을 선택한다.

**입력 항목**:

- **최종 콘텐츠 주제** — 한 문장으로 정의  
  - 힌트: "나는 [누구]에게 [어떤 문제]를 해결하는 [콘텐츠 형식]을 만든다"
- **템플릿 형식 선택** — 시각 카드 선택 UI

| 형식 | 설명 |
|---|---|
| 카드뉴스 | SNS용 정보 전달 카드 (4~8컷) |
| 숏폼 영상 | 릴스·쇼츠·틱톡용 60초 이내 영상 |
| 4컷 만화 | 스토리텔링형 일러스트 콘텐츠 |
| 랜딩페이지 | 캠페인·이벤트용 단일 페이지 |
| 인포그래픽 | 데이터·정보 시각화 이미지 |

**핵심 동작**: 템플릿 선택 즉시 Step 3 입력 필드가 동적으로 변경됨

---

### Step 03 · 상세 구성 및 제작 가이드

**목적**: 선택한 템플릿에 맞춰 콘텐츠의 세부 요소를 결정한다.

**공통 입력 항목**:

- 타겟 독자 (연령대, 관심사, 상황)
- 톤앤매너 선택 칩 (정보형 / 감성형 / 유머형 / 신뢰형 / 과몰입형)
- 이미지 스타일 선택 칩 (일러스트 / 실사 / 플랫디자인 / 타이포그래피 중심)

**템플릿별 추가 입력 항목**:

#### 카드뉴스
- 총 컷 수 (4 / 6 / 8컷)
- 표지 메인 카피
- 각 컷 소제목 + 본문 내용
- 마지막 컷 CTA (Call to Action)

#### 숏폼 영상
- 오프닝 후크 (첫 3초 자막/멘트)
- 핵심 메시지 (15~30초 구간)
- 클로징 CTA + 음악 분위기

#### 4컷 만화
- 캐릭터 성격 선택 (귀여움 / 공감형 / 유머러스 / 진지함)
- 1~4컷 상황 설명
- 대사 초안

**AI 보조**: 각 항목 옆 "AI로 채우기" 버튼으로 빈칸 자동 완성  
**UI 패턴**: `feature-list` 형태 + 동적 필드 렌더링

---

### Step 04 · 스토리보드 미리보기

**목적**: 입력한 내용을 컷 단위로 시각화하여 전체 흐름을 한눈에 확인한다.

**출력 형식**:

- 컷 카드 나열 (좌→우 또는 상→하)
- 각 컷 카드 구성:
  - 컷 번호 + 제목
  - 이미지 설명 (AI 이미지 생성에 사용될 내용)
  - 카피/대사
  - 예상 화면 레이아웃 스케치 (텍스트 기반)

**차별화 포인트**: 바이브코딩 앱의 "화면 구성" 스텝과 동일한 역할 — 제작 전 전체 흐름 검토 가능

**UI 패턴**: `screens-grid` 카드 레이아웃 확장 적용

---

### Step 05 · 프롬프트 생성 및 기획서 완성

**목적**: 실제 콘텐츠 제작에 바로 사용할 수 있는 결과물을 모두 생성한다.

**출력 결과물**:

1. **마케팅 PRD 기획서** — 전체 기획 내용을 문서 형태로 정리
2. **AI 이미지 생성 프롬프트** — 컷별 이미지 생성용 (Midjourney / DALL-E / Stable Diffusion 형식)
3. **카피라이팅 프롬프트** — Claude / ChatGPT 용 카피 생성 프롬프트
4. **스토리보드 전체 보기** — 인쇄·공유 가능한 형태

**내보내기 옵션**:

| 기능 | 과금 여부 |
|---|---|
| 기획서 텍스트 복사 | 무료 |
| 기획서 PDF 저장 | 무료 |
| 스토리보드 이미지 다운로드 | **유료** |
| AI 프롬프트 패키지 다운로드 | **유료** |

## 핵심 기술 구조

### 아키텍처

```
단일 HTML 파일 (Single-file)
├── CSS (인라인 스타일 + CSS 변수)
├── HTML (5개 step-panel)
└── Vanilla JS
    ├── goStep() — 단계 이동 + 유효성 검사
    ├── renderStep3Fields() — 템플릿별 동적 필드 생성
    ├── generateStoryboard() — 스토리보드 카드 렌더링
    ├── generatePrompts() — AI 프롬프트 자동 생성
    └── generatePRD() — 최종 기획서 조합 출력
```

### AI 연동 (선택적)

- 각 단계 "AI로 채우기" 버튼: Gemini API 또는 Claude API 호출
- Step 1 URL 요약: 웹 리서치 결과 요약
- Step 3 카피 초안: 입력 정보 기반 자동 생성

### 데이터 저장

- `localStorage` 활용 — 페이지 새로고침 시 입력값 유지
- 최종 결과물: JSON 직렬화 후 텍스트/PDF 출력

## 바이브코딩 앱에서 재사용할 컴포넌트

기존 `vibe-coding-planner.html` 코드에서 그대로 가져올 수 있는 요소들:

- **스텝 진행 바** — `.progress-bar`, `.prog-step` 구조
- **힌트 박스** — `.hint-box` 패턴
- **선택 칩** — `.sel-option`, `.tag` 컴포넌트
- **기능 목록** — `.feature-list`, `.feature-item` 패턴
- **화면 카드 그리드** — `.screens-grid`, `.screen-card` 패턴
- **단계 이동 로직** — `goStep()`, 유효성 검사 함수
- **최종 출력 로직** — 기획서 조합 함수 (마케팅 PRD 버전으로 교체)

- **무료**: 기획서 열람, 텍스트 복사, PDF 저장
